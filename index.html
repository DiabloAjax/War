How to use

Purpose

This app is designed for assigning players to 12 rooms, quickly adjusting rosters, searching, importing lists, and exporting the result as text or an image. All changes are saved automatically and restored the next time the file is opened.

Screen layout

On desktop, the free player pool is on the left and the rooms are on the right. The pool has its own search, upload button, help button, and hide button. The rooms area has its own search, a copy-all button, and a clean snapshot button. On mobile, the pool opens as a bottom modal by pressing Player pool.

Working with players

Players can be dragged from the pool into rooms, between rooms, and back into the pool. Dropping one player onto another inside a room swaps them. The ↩ button on a room card immediately sends the player back to the pool.

Quick actions

Double-clicking a player card in the pool sends that player to the next room in order. Double-clicking a player inside a room returns them to the pool. While dragging, page auto-scroll works, and the pool has its own internal auto-scroll.

Editing a player

Press ✎ on a card to open editing. In edit mode you can change the player name and choose one of 7 standard colors. Save with ✓ or Enter. Empty names are not allowed. While a card is being edited, double-click shortcuts and dragging are disabled for that card.

Quick move from the dropdown

Next to the edit button on each card there is a room dropdown. Selecting a room number moves the player there immediately. This works both for players in the pool and for players already inside rooms.

Search

Room search and pool search work independently. Matches are highlighted directly inside the player name. Search also supports approximate latin/cyrillic input. When you type, the interface scrolls to the first match. The × button clears search, and Esc clears the active field.

Rooms and occupancy

Each room has a visual occupancy state: empty, partially filled, nearly full, and full. The counter shows current/8. If there are fewer than 8 players, the current number turns red. Overflow is also allowed: players can be moved into already full rooms, so the counter may become 9/8 or more.

Room anchors

On mobile, fixed room anchors appear on the left: 1-1, 1-2, and so on. Tapping an anchor scrolls the page to that room. The anchor color shows room status: empty is gray, full is green, overfilled is red.

Import from Excel and CSV

The Upload button opens a file picker. The app reads the first sheet and the first column. If a name already exists, that player stays unchanged. If a name does not exist, a new player is created in the pool. If the app currently contains players that are missing from the imported file, they are removed entirely, including from rooms. New imported players get orange as the default color.

Copying rosters

The ⧉ button in the top toolbar copies all rooms at once. The ⧉ button inside a room copies only that room. When copied, names are automatically converted to the @nickname format without spaces. After copying, the button and the corresponding block briefly highlight. On mobile, a stronger clipboard fallback is used.

Clean snapshot

The ✦ button creates a clean PNG snapshot of the rooms area. In export mode the app removes service UI elements such as search, card buttons, dropdowns, and extra controls. Room headers on the snapshot keep only the room number without the word Room.

Pool on mobile

On phones, the pool opens from the bottom Player pool button. The pool has its own internal scroll. To start dragging a player, do not just swipe; hold the card briefly. Once the hold becomes a real drag, the app gives a light vibration, the pool moves off-screen, and a ghost card appears under your finger. The room under your finger is highlighted more strongly to make dropping easier.

Hiding the pool and saving

On desktop, the ‹ button hides the pool and › brings it back. When the pool is hidden, the rooms area expands. All actions are saved automatically: moves between rooms, reordering, name edits, color changes, file imports, and overfilled rooms. When the page is reopened, the last saved state is restored.

Hotkeys

Enter saves the player name while editing. Esc closes the help window or clears the active search field. Most other actions are performed with mouse or touch: drag-and-drop, double-click, card buttons, and the room dropdown.

Choose a settings file

To enable autosave, choose an existing file or create a new one.
